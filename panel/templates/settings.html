{{ icon('shield', 18) }} AI از مسیر گوگل — روش امن

{% if forwarder.enabled %}فعال {% else %}غیرفعال{% endif %}
{{ icon('info', 16) }}
این روش، Domain Fronting را حفظ می‌کند. مسیر:
browser → relay → TLS(SNI=google.com) → Apps Script → Worker → HTTPS → VPS شما → مقصد
ISP فقط یک اتصال TLS به گوگل می‌بیند، و مقصد IP ثابت VPS شما را می‌بیند (نه رنج Cloudflare که OpenAI بلاک کرده). این تفاوت اصلی با روش پایین است.
روی VPS اجرا کنید — با دامنه 
curl -fsSL https://raw.githubusercontent.com/ArasTey/Aras-GP/main/scripts/install-forwarder.sh -o f.sh
sudo bash f.sh --domain fwd.example.com
یا بدون دامنه — با Cloudflare Tunnel 
curl -fsSL https://raw.githubusercontent.com/ArasTey/Aras-GP/main/scripts/install-forwarder.sh -o f.sh
sudo bash f.sh --tunnel
هر دو مقدار را اسکریپت نصب آخر کارش چاپ می‌کند. «تست» یک درخواست واقعی از فورواردر می‌فرستد و IP خروجی را نشان می‌دهد.
هر دامنه در یک خط؛ زیردامنه‌ها خودکار شامل می‌شوند.
{{ icon('alert', 16) }}
بعد از ذخیره، حتماً یک بار Worker را دوباره دیپلوی کنید (صفحه‌ی دیپلوی → دیپلوی Worker). آدرس و کلید فورواردر باید روی خود Worker بایند شوند وگرنه فوروارد انجام نمی‌شود.

چرا ChatGPT با رله باز نمی‌شود؟ ترافیکی که از Cloudflare Workers خارج می‌شود با IP رنج کلودفلر به مقصد می‌رسد و سرویس‌هایی مثل OpenAI آن رنج را مسدود کرده‌اند — همان خطای «Unable to load site» با Ray ID. این داخل رله قابل حل نیست؛ راهش یک خروجی دیگر است.

اسکریپت زیر را روی هر VPS ارزانی اجرا کنید تا یک SOCKS5 با رمز بالا بیاورد. همان لحظه یک خط به شما می‌دهد که در کادر پایین می‌گذارید. از آن به بعد فقط میزبان‌های فهرست‌شده از VPS شما خارج می‌شوند — یعنی هم AI کار می‌کند، هم آن دامنه‌ها IP ثابت می‌گیرند.
روی VPS اجرا کنید 
curl -fsSL https://raw.githubusercontent.com/ArasTey/Aras-GP/main/scripts/install-exit-node.sh | sudo bash
همان خطی که اسکریپت نصب چاپ می‌کند. «تست» یک درخواست واقعی می‌فرستد و IP خروجی را نشان می‌دهد.
هر دامنه در یک خط. زیردامنه‌ها خودکار شامل می‌شوند (openai.com یعنی chat.openai.com هم).
